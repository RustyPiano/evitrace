## M4-06 Skill 与健康页面
- `views/AdminSkillsView.vue`：名称/版本/required/enabled/最近健康状态；required 开关禁用（不可停用），非 required 可启停（`PATCH /admin/skills/{id}`）；可触发健康探测（`POST /admin/skills/{id}/health`）。
- `views/AdminHealthView.vue`：展示组件健康——数据库 / 磁盘可写 / 本地模型(LLM) / FFmpeg / OCR / ASR。组件不可用显示错误原因；**页面不得暴露 Secret 或完整磁盘路径**。
- `views/AdminAuditView.vue`：审计日志列表（分页）。
- **后端补齐**（若尚不完整）：
  - `GET /api/v1/admin/health`：综合组件健康（db 探测；disk 可写+大致可用性，不暴露完整路径；llm 用轻量探活，MOCK 下标 healthy/skipped；ffmpeg=`shutil.which`；ocr/asr=`find_spec`+模型目录是否就绪/或 MOCK 模式）。返回结构化 `{component, status: healthy/unavailable/skipped, detail(脱敏)}`。
  - `GET /api/v1/admin/audit-logs`：管理员分页查询审计日志。
  - 确认审计已覆盖 SPEC §7.8/§12 要求：登录成功/失败、创建/删除任务、上传文件、**启动分析**、**下载报告**、创建/停用用户、**启停 Skill**。缺的补上（detail 脱敏）。
