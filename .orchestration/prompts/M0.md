# 任务：实现 EviTrace 里程碑 M0（项目骨架与运行环境）

你是本项目的实现工程师。项目根目录已有 `SPEC.md` 和 `PLAN.md`，它们是唯一权威规格。

## 开始前必读
1. 阅读 `PLAN.md` 第 2 章（M0：M0-01 ~ M0-04）以及第 0 章开发规则、第 11 章禁止事项。
2. 阅读 `SPEC.md` 第 6 章（系统架构/技术栈/目录）、第 16 章（建议仓库结构）。

## 本里程碑必须交付（只做 M0，不要提前做 M1+ 的业务逻辑）

### M0-01 仓库结构
- 创建目录：`backend/`、`frontend/`、`demo_data/`、`scripts/`。
- 根目录 `README.md`：至少含项目简介与开发启动占位说明（后续里程碑会扩充）。
- 根目录 `.env.example`：必须**完全包含** PLAN.md M0-01 列出的全部环境变量键（APP_NAME、ENV、SECRET_KEY、ACCESS_TOKEN_EXPIRE_HOURS、DATABASE_URL、DATA_ROOT、MAX_UPLOAD_MB、CORS_ORIGINS、LOCAL_LLM_BASE_URL、LOCAL_LLM_API_KEY、LOCAL_LLM_MODEL、LLM_TIMEOUT_SEC、LLM_MAX_RETRIES、MOCK_AI、VIDEO_FRAME_INTERVAL_SEC、TIME_CONFLICT_MINUTES、FIRST_ADMIN_USERNAME、FIRST_ADMIN_PASSWORD），值用占位/示例值，不得包含真实 secret。
- 根目录 `.gitignore` 已存在，可按需补充（务必忽略 `data/`、`__pycache__/`、`node_modules/`、`*.pyc`、`dist/`、虚拟环境目录）。

### M0-02 后端骨架
按 SPEC 第 16 章结构创建：`backend/app/main.py`、`config.py`、`database.py`、`models.py`、`schemas.py`、`backend/requirements.txt`。
- FastAPI app，统一前缀 `/api/v1`。
- 健康接口 `GET /api/v1/health` 返回 200 与简单 JSON。
- 配置加载用 `pydantic-settings`，从环境变量/`.env` 读取，配置缺失时启动错误信息明确。
- SQLite：`DATABASE_URL` 默认 sqlite，启动时自动创建 `DATA_ROOT`/data 目录与数据库文件，启动时建表（M0 阶段可只有空的 Base/metadata，`models.py` 留最小占位，完整表在 M1）。
- 配置 CORS，仅允许 `CORS_ORIGINS`。
- 统一错误处理框架：实现 SPEC 第 8 章的统一错误响应结构 `{"detail":{"code":..., "message":...}}`，并提供一个自定义异常基类供后续里程碑使用（如 `AppError(code, message, http_status)`）。
- `requirements.txt`：包含 M0 必需依赖（fastapi、uvicorn[standard]、pydantic、pydantic-settings、SQLAlchemy 2、python-multipart）。**重要：把重型可选依赖（PyMuPDF、python-docx、paddleocr、faster-whisper、Pillow 等）单独放，并确保应用在它们未安装时仍能 import 和启动**——这些只在对应 Skill 内部延迟 import（M2 才用）。M0 不要 import 它们。

### M0-03 前端骨架
在 `frontend/` 创建 Vue 3 + TypeScript + Vite 项目：
- 集成 Vue Router、Pinia、Axios、Element Plus。
- 基础 Layout 组件。
- 路由：`/login` 与 `/tasks` 两个空白占位页面（可显示标题即可）。
- Axios 实例：base URL 指向 `/api/v1`（生产用相对路径，开发用 Vite proxy），并预置 JWT 请求拦截器（从 store/localStorage 读取 token）与 401 响应处理占位。
- `vite.config.ts` 开发代理：`/api` → 后端 `http://localhost:8000`。
- 不得加载任何 CDN 资源（离线要求 NFR-001），依赖均走 npm。
- TypeScript 严格模式，确保 `npm run type-check`/`npm run build` 无错误（package.json 中提供 `type-check` 脚本，如用 vue-tsc）。

### M0-04 Docker 开发环境
- `backend/Dockerfile`（基于 python:3.11-slim）、`frontend/Dockerfile`、根目录 `docker-compose.yml`。
- compose 映射数据目录为 volume（持久化 `data/`）。
- 前端容器代理 `/api` 到后端容器。
- 不把模型权重打入镜像。
